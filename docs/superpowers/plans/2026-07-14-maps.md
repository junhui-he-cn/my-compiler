# Maps / Dictionaries Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement first-class ordered map values with `map<K, V>` typing, index lookup/upsert, `len`, `contains`, and C++/`.cdbc`/Rust VM parity without adding map iteration.

**Architecture:** Map construction is a new variable-arity IR/bytecode operation whose operands are alternating key/value registers. Existing `Index`, `AssignIndex`, and `Len` operations dispatch on arrays or maps at runtime; `contains` remains a shadowable native call with unshadowed member sugar. Rust stores an insertion-ordered shared entry vector so runtime equality and output stay deterministic across the C++ emitter and Rust VM.

**Tech Stack:** C++17 compiler front end and register IR, C++ bytecode/text emitter, Python golden/artifact runners, Rust 2021 bytecode parser and VM, CMake/CTest, Cargo.

## File Map and Ownership

- `include/Ast.hpp`, `src/Ast.cpp`: map type-annotation and expression nodes plus stable AST printing.
- `include/Parser.hpp`, `src/Parser.cpp`: `{ key: value }` expression parsing and `map<K, V>` annotations.
- `include/TypeUtils.hpp`, `src/TypeUtils.cpp`: `StaticType::Map`, map component metadata, names, compatibility, and type construction.
- `include/TypeChecker.hpp`, `src/TypeChecker.cpp`: map inference, annotation resolution, index/assignment rules, `len`/`contains`, and array-only `for-in` enforcement.
- `include/IR.hpp`, `src/IR.cpp`, `include/IRCompiler.hpp`, `src/IRCompiler.cpp`: map IR construction, printing, and AST lowering.
- `include/Bytecode.hpp`, `src/Bytecode.cpp`, `src/BytecodeCompiler.cpp`, `src/BytecodeTextEmitter.cpp`: map bytecode operation and `.cdbc` emission.
- `vm-rs/src/bytecode.rs`, `vm-rs/src/format.rs`: Rust map instruction representation, parser, and canonical formatter.
- `include/Value.hpp`, `src/Value.cpp`, `vm-rs/src/runtime.rs`, `vm-rs/src/value.rs`, `vm-rs/src/vm.rs`: map values, identity/equality/formatting, lookup/upsert, length, and native dispatch.
- `tests/type_utils_tests.cpp`, `vm-rs/src/format.rs` tests, `vm-rs/src/vm.rs` tests: focused unit coverage.
- `tests/golden/`, `tests/bytecode_artifacts/`: parser/type/runtime/IR/bytecode/Rust parity fixtures.
- `docs/language-grammar.ebnf`, `README.md`, `docs/bytecode-text-format.md`, `docs/roadmap.md`, `AGENTS.md`: user and project-memory documentation.

Each task below ends with a focused test run and a small commit. Keep map `for-in`, deletion, custom iterators, structural map equality, and higher-order map helpers out of every task.
