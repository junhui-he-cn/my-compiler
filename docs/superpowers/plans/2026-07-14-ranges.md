# Ranges Implementation Plan

**Goal:** Implement immutable finite integer ranges with native construction,
static typing, indexed access, membership, length, and array-style `for-in`
execution across the C++ compiler and Rust VM.

**Architecture:** Keep range construction on the existing shadowable native-call
path. Add a scalar `Range` type to C++ and Rust, generalize existing collection
checks, and preserve the `assert_array` artifact spelling while accepting both
arrays and ranges at runtime.

## Tasks

1. Add `StaticType::Range`, range native signature metadata, type-checking for
   one-to-three numeric arguments, range-aware `len`/`contains`/index rules,
   and focused type utility tests.
2. Add C++ `RangeValue` representation and update constant/printer switches;
   extend existing `AssertArray` lowering/printing only where needed to keep
   range-aware IR and bytecode output stable.
3. Add Rust `RangeValue`, numeric bound validation, half-open length/value
   calculations, `range` native execution, structural equality/formatting,
   range-aware `len`, `Index`, `contains`, and `AssertArray` execution.
4. Add Rust unit tests and end-to-end fixtures for ascending/descending/empty
   ranges, indexing, membership, `len`, `for-in`, type errors, and runtime
   validation errors. Refresh only affected goldens and artifacts.
5. Update README, AGENTS.md, roadmap status, and bytecode-format notes; run
   the CMake, Python, artifact, Rust VM, and Cargo verification commands.
