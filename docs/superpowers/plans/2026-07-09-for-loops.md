# C-Style For Loops Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add C-style `for [initializer]; [condition]; [increment] { ... }` loops with existing truthiness, lexical scoping, `break`, `continue`, IR execution, bytecode emission, and Rust VM parity.

**Architecture:** Add a `For` token and `ForStmt` AST node, parse a three-clause loop header with optional pieces, and type-check it in a loop-local scope. Lower `ForStmt` directly to existing IR jumps so `continue` targets the increment expression before the next condition check; bytecode and Rust VM continue to use existing control-flow opcodes.

**Tech Stack:** C++17 lexer/parser/AST/type checker/IR compiler, Python golden tests, bytecode artifact tests, Rust VM parity tests, Markdown/EBNF docs.

---

## File Structure

- `include/Token.hpp`, `src/Lexer.cpp`: add `For` keyword token.
- `include/Ast.hpp`, `src/Ast.cpp`: add `ForStmt` with optional initializer, condition, increment, and block body printing.
- `include/Parser.hpp`, `src/Parser.cpp`: parse `for` statements and helper routines for initializer clauses that do not consume the header semicolon twice.
- `src/TypeChecker.cpp`: type-check for-loop scope and loop-depth behavior.
- `include/IRCompiler.hpp`, `src/IRCompiler.cpp`: add `compileFor()` and lower continue to the increment target.
- `docs/language-grammar.ebnf`, `README.md`, `docs/roadmap.md`, `AGENTS.md`: document C-style for loops and defer `for-in`.
- `tests/golden/`, `tests/bytecode_artifacts/`, `tests/run_rust_vm_tests.py`: add success, parse-error, type-error, artifact, and parity coverage.

```cpp
StmtPtr Parser::forStatement()
{
    Token keyword = previous();

    StmtPtr initializer = forInitializer();
    consume(TokenType::Semicolon, "expected `;` after for initializer");

    ExprPtr condition;
    if (!check(TokenType::Semicolon)) {
        condition = expression();
    }
    consume(TokenType::Semicolon, "expected `;` after for condition");

    ExprPtr increment;
    if (!check(TokenType::LeftBrace)) {
        increment = expression();
    }
    consume(TokenType::LeftBrace, "expected `{` after for clauses");
    StmtPtr body = blockStatement();

    return std::make_unique<ForStmt>(std::move(keyword), std::move(initializer), std::move(condition), std::move(increment), std::move(body));
}
```

```cpp
void IRCompiler::compileFor(const ForStmt& statement)
{
    if (statement.initializer) {
        compileStatement(*statement.initializer);
    }

    const std::size_t loopStart = ir_.instructionCount();
    std::size_t exitJump = static_cast<std::size_t>(-1);
    if (statement.condition) {
        const IRRegister condition = compileExpression(*statement.condition);
        exitJump = ir_.emitJumpIfFalse(condition);
    }

    const std::size_t jumpOverIncrement = ir_.emitJump();
    const std::size_t incrementStart = ir_.instructionCount();
    if (statement.increment) {
        compileExpression(*statement.increment);
    }
    ir_.emitJumpTo(loopStart);

    ir_.patchJump(jumpOverIncrement);

    loopContexts_.push_back(LoopContext{incrementStart, {}});
    compileStatement(*statement.body);
    LoopContext loop = std::move(loopContexts_.back());
    loopContexts_.pop_back();

    ir_.emitJumpTo(incrementStart);
    if (exitJump != static_cast<std::size_t>(-1)) {
        ir_.patchJump(exitJump);
    }
    for (const std::size_t breakJump : loop.breakJumps) {
        ir_.patchJump(breakJump);
    }
}
```
