# Closures Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement Phase 6B closures for existing nested `fun` declarations with by-reference capture of enclosing local variables.

**Architecture:** Preserve compile-time lexical name resolution and the existing IR function table. Change runtime variable storage from raw `Value` maps to shared variable cells, extend function values with a captured environment and identity, and make `make_function` capture the current local/closure cells. Function declarations create a self-binding placeholder before `make_function` so nested recursive closures can capture themselves.

**Tech Stack:** C++17, CMake, recursive-descent parser, resolver/type checker, register IR, IR interpreter call frames, Python golden tests, CTest.

---

## File Structure

- Modify: `include/Value.hpp`, `src/Value.cpp` — add `Cell`, `Environment`, `FunctionValue::closure`, and function identity equality.
- Modify: `include/IR.hpp`, `src/IR.cpp` — support nested IR function building with a function context stack so nested `fun` declarations can compile while an outer function body is active.
- Modify: `include/IRInterpreter.hpp`, `src/IRInterpreter.cpp` — replace value maps with cell environments; capture closure cells in `MakeFunction`; call functions with closure frames; preserve `globals()` API through a value snapshot.
- Modify: `src/IRCompiler.cpp` — emit function declaration self-binding placeholder before `make_function`, then assign the actual function value into that cell.
- Modify: `src/TypeChecker.cpp` — allow local captures by removing the Phase 6A no-capture checks.
- Modify/Create/Remove: `tests/golden/function_*`, `tests/golden/type_errors/function_capture_local.*` — add closure success coverage and migrate old no-capture error.
- Modify: `README.md`, `docs/roadmap.md`, `AGENTS.md` — document closures and add Phase 6C function expressions/lambdas to roadmap.
- Verify: `docs/language-grammar.ebnf` — no grammar change is expected because closures use existing `fun` syntax.

```cpp
std::shared_ptr<Cell> IRInterpreter::findCell(const Frame& frame, const std::string& name) const
{
    if (frame.locals) {
        const auto local = frame.locals->values.find(name);
        if (local != frame.locals->values.end()) {
            return local->second;
        }
    }

    if (frame.closure) {
        const auto captured = frame.closure->values.find(name);
        if (captured != frame.closure->values.end()) {
            return captured->second;
        }
    }

    if (globals_) {
        const auto global = globals_->values.find(name);
        if (global != globals_->values.end()) {
            return global->second;
        }
    }

    return nullptr;
}

Value IRInterpreter::loadVariable(const Frame& frame, const std::string& name) const
{
    std::shared_ptr<Cell> cell = findCell(frame, name);
    if (!cell) {
        throw IRRuntimeError("undefined variable `" + name + "`");
    }
    return cell->value;
}

void IRInterpreter::storeVariable(Frame& frame, const std::string& name, Value value, bool isMain)
{
    std::shared_ptr<Cell> cell = std::make_shared<Cell>(std::move(value));
    if (isMain) {
        globals_->values.insert_or_assign(name, std::move(cell));
        return;
    }
    frame.locals->values.insert_or_assign(name, std::move(cell));
}

void IRInterpreter::assignVariable(Frame& frame, const std::string& name, Value value)
{
    std::shared_ptr<Cell> cell = findCell(frame, name);
    if (!cell) {
        throw IRRuntimeError("undefined variable `" + name + "`");
    }
    cell->value = std::move(value);
}
```

- [ ] **Step 8: Add capture and globals snapshot helpers**

Add these definitions after `assignVariable()`:

```cpp
std::shared_ptr<Environment> IRInterpreter::captureEnvironment(const Frame& frame) const
{
    auto captured = std::make_shared<Environment>();

    if (frame.closure) {
        for (const auto& [name, cell] : frame.closure->values) {
            captured->values.insert_or_assign(name, cell);
        }
    }

    if (frame.locals) {
        for (const auto& [name, cell] : frame.locals->values) {
            captured->values.insert_or_assign(name, cell);
        }
    }

    return captured;
}

void IRInterpreter::refreshGlobalsView() const
{
    globalsView_.clear();
    if (!globals_) {
        return;
    }
    for (const auto& [name, cell] : globals_->values) {
        globalsView_.emplace(name, cell->value);
    }
}
```

This intentionally copies closure cells first and current locals second so locals shadow outer captured names in the captured environment.

Edit `IRCompiler::compileFunctionStatement()` in `src/IRCompiler.cpp`.

Replace the method body with:

```cpp
void IRCompiler::compileFunctionStatement(const FunctionStmt& function)
{
    const std::string functionName = resolvedNames_->functionName(function);
    IRRegister placeholder = ir_.emitConstant(Value::nil());
    ir_.emitStoreVar(functionName, placeholder);

    std::vector<std::string> parameters = resolvedNames_->parameterNames(function);
    ir_.beginFunction(function.name.lexeme, std::move(parameters));

    for (const auto& statement : function.body) {
        compileStatement(*statement);
    }
    IRRegister nilValue = ir_.emitConstant(Value::nil());
    ir_.emitReturn(nilValue);

    const std::size_t functionIndex = ir_.endFunction();
    IRRegister value = ir_.emitMakeFunction(functionIndex);
    ir_.emitAssignVar(functionName, value);
}
```

This creates a cell for the function binding before `make_function` captures the current environment, then mutates that cell to hold the actual function value.
